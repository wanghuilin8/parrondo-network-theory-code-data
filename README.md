# Code and Data for Networked Parrondo Games

This repository contains the code, cached numerical arrays, CSV outputs, and
generated figure files associated with the manuscript:

**Analytical Theory of Parrondo's Paradox on Complex Networks: Necessary
Condition, Critical Penalty, and Topological Robustness**

Repository URL:
<https://github.com/wanghuilin8/parrondo-network-theory-code-data>

No external experimental or observational datasets are used. The numerical
data in this repository are generated by the analytical, simulation, and
figure-generation scripts listed below.

## Contents

- `generate_figures.py`: regenerates the publication-style figures from the
  complete one-node analytical calculations, cached stationary Monte Carlo
  arrays, and simulation outputs.
- `run_robustness_simulations.py`: runs the additional robustness simulations
  used for Fig. 7 and writes CSV/text summaries to `data/`.
- `test_neighbor_sampling.py`: verifies that Game A samples a neighbor
  uniformly from the focal node's actual neighbor list.
- `requirements.txt`: Python dependencies.
- `data/figure_data_cache.npz`: cached numerical arrays used by the
  figure-generation script. The cache version is 23.
- `data/additional_robustness_curves.csv`: CSV output for the additional
  robustness checks.
- `data/additional_robustness_summary.txt`: text summary of the additional
  robustness windows.
- `figures/`: generated figure files used in the revised manuscript.

## Model and Simulation Protocol

At each update, a focal node is selected uniformly. With probability `gamma`,
Game A is played: the focal node samples one of its actual neighbors uniformly,
receives a capital transfer `+1` or `-1` with equal probability, and
independently pays an environmental penalty `-1` with probability `tau`.
With probability `1-gamma`, Game B is played: if the focal node is in state
`c_i mod 3 = 0`, it wins with probability `p1`; otherwise it wins with
probability `p2`.

Stationary Monte Carlo measurements use a separate burn-in phase followed by
a measurement phase. In the publication scripts, the main WS/BA cached
figures use 100 burn-in updates per node and 100 measurement updates per node.
The additional Fig. 7 robustness simulations use 100 burn-in updates per node
followed by 100 measurement updates per node.

## Reproducing the Results

Use Python 3.10 or newer. From the repository root, run:

```bash
python -m pip install -r requirements.txt
python test_neighbor_sampling.py
python run_robustness_simulations.py
python generate_figures.py
```

The simulation scripts fix the random seeds. Monte Carlo fitness is accumulated
only after a burn-in phase, and stationary trap occupancy is sampled after
burn-in. In each Game A update, the focal node selects one of its actual
neighbors uniformly at random.

The submitted manuscript can be reproduced directly from the cached numerical
data in `data/figure_data_cache.npz` and the CSV/text outputs in `data/`.
The figure-generation script writes the complete figure set to `figures/`.
